# D-20260714-02 — Completion ledger is an open-work queue

Index: [DecisionHistory.md](../DecisionHistory.md)

## Original record

Original heading: 2026-07-14 - Completion ledger became an open-work queue

Decision: `CompletionLedger.md` contains only active unresolved work. An item
is removed in the same change that implements and verifies it, and the ledger
is deleted when no active items remain. Version control is the default
completion history. Consequential decisions remain in `DecisionHistory.md`.
`CompletionHistory.md` is created only for an explicit audit-retention need and
is excluded from routine working context.

Why: Retaining resolved rows mixed queue state with historical evidence, made
ordinary context grow indefinitely, and allowed stale closed work to obscure
the actual readiness blockers. The six existing rows were already preserved in
version control, while their consequential outcomes were recorded here where
applicable, so copying them to another permanent ledger would add no evidence.

Result: The universal policy now defines the full open-only lifecycle. Its
semantic checker rejects retained terminal history in either term order,
negative or delayed removal, and an empty kept ledger while distinguishing the
cold history destinations. A separate repository-state checker requires one
documented active-work table and rejects terminal or unknown status rows, mixed
active/terminal states, content outside the schema, duplicate IDs, and
header-only ledgers while allowing active rows whose verification prose merely
mentions resolution. The resolved-only repository ledger is removed after the
new guard proves it detects that exact state.
